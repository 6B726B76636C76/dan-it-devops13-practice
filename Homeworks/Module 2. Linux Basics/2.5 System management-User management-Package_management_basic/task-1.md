1 - 
    1 - После включения устройство начинает поиск загрузочного устройства.
    2 - Запускается первичный загрузчик, расположенный в первых секторах памяти загрузочного устройства.
    3 - Далее запускается основной загрузчик, который позволяет выбрать ядро, инициализирует файловую систему и передаёт параметры запуска ядру.
    4 - Основной загрузчик передаёт управление системной ядру. Происходит инициализация процессора, памяти и interrupts
    5 - Ядро создаёт временную корневую файловую систему и загружает модули, далее происходит монтирование корневой директории /
    6 - Ядро переключается со временной корневой файловой системы на реальный root
    7 - Ядро запускает systemd с PID=1, начинается параллельный запуск служб системы
    8 - Systemd начинает монтирование всех необходимых файловых систем из /etc/fstab, запуск всех необходимых служб. (/etc/fstab это конфигурационный файл, который определяет какие фс и как должны в системе монтироваться). Без fstab система загрузится, но не примонтируются дополнительные диски, может не включиться swap.
    9 - Запускается getty или дисплейный менеджер (GNOME/KDE/XFCE etc) и открывается экран входа в систему

3 - 
     -rw-------
     Это разрешение позволяет только владельцу файла его читать и записывать в него данные. Общая схема прав такая(owner-group-other_users) 1-2-3, где 1 - права владельца, 2 - права группы, 3 - права других пользователей, не входящих в группу, указанную в разрешении.
     Соответственно, в нашем случае 2 и 3 не имеют никаких прав и доступ есть только у владельца файла. Чтоб добавить право на executable, необходимо добавить флаг x.
     Дальше уже зависит от владельца файла. Если локальный юзер, то делаем просто chmod +x <file>, если надо добавить для всех, если же только для владельца файла, то проще сделать chmod 700 <file>. Для файлов, у которых владелец root необходимо право добавлять через sudo (sudo chmod +x <file> или sudo chmod 700 <file>)

4 - 
    Разница между apt и dpkg в том, что apt разрешает необходимые зависимости при установке пакета, в то время как dpkg этого не делает. Через dpkg удобнее работать с уже установленными пакетами в системе
